Detailed Description

Main flight controller application for STM32F411-based quadcopter.

This module implements the main control loop and state machine for a quadcopter flight controller. It coordinates IMU sensor readings, PID control loops, motor control, and LoRa wireless communication.

The controller operates at 6660 Hz IMU update rate and implements:

  • Multi-state flight control (disarmed, arming, flying, manual, emergency stop)
  • Attitude stabilization using PID control on pitch, roll, and yaw
  • Velocity-based position corrections
  • Heartbeat-based safety monitoring with automatic emergency stop
  • LoRa-based command and telemetry communication
  • Manual motor control mode for testing and calibration

Motor Layout (X configuration): Front 4 2 \ + / |/ /|\ / - \ 1 3 Rear

Motors 1,4: CW rotation (create CCW torque) Motors 2,3: CCW rotation (create CW torque)

Compute motor speeds and send throttle commands to motors.

Implements the motor mixing algorithm to combine base throttle, PID corrections, and per-motor biases into individual motor throttle values. The mixing accounts for the quadcopter X configuration and motor rotation directions.

Parameters
base_throttleBase throttle level (0.0-1.0) before corrections
pidPointer to PID structure containing pitch/roll/yaw corrections
biasPointer to MotorBias structure with per-motor trim values

Implements motor mixing for X-configuration quadcopter. Combines base throttle, PID corrections (pitch, roll, yaw), and individual motor biases to compute final throttle values for each motor.

Motor Mixing Algorithm (X configuration):

  • Motor 1 (rear left, CW): throttle - pitch + yaw + bias1
  • Motor 2 (front right, CCW): throttle + roll - yaw + bias2
  • Motor 3 (rear right, CCW): throttle - roll - yaw + bias3
  • Motor 4 (front left, CW): throttle + pitch + yaw + bias4

Pitch correction: Motor 4 (front) increases, Motor 1 (rear) decreases Roll correction: Motor 2 (right) increases, Motor 3 (left) decreases Yaw correction: CW motors (1,4) increase, CCW motors (2,3) decrease

Final throttle values are clamped to MIN_THROTTLE-MAX_THROTTLE range and converted to DSHOT protocol values (1-1001 range).

Parameters
base_throttleBase throttle level (0.0-1.0) before corrections
pidPointer to PID structure containing pitch/roll/yaw output corrections
biasPointer to MotorBias structure with per-motor trim values (0.0-1.0)

◆ main()

int main ( void )

Main entry point and control loop for flight controller.

Initializes all subsystems and runs the main state machine loop. The state machine coordinates system initialization, IMU updates, PID control, motor control, and wireless communication.

State Flow:

  • INIT: Configure clocks, peripherals, sensors, and communication
  • DISARMED: Wait for commands, allow configuration changes
  • CALIBRATE: Perform IMU gyro/accel bias calibration
  • ARMING/ARMING_MANUAL: Initialize motors and transition to active mode
  • FLYING: Run high-speed control loop with IMU (6660 Hz) and PID updates
  • MANUAL: Direct motor control mode for testing
  • EMERGENCY_STOP: Safety state requiring manual reset

Safety Features:

  • Heartbeat timeout monitoring (2 second timeout triggers emergency stop)
  • Max angle protection (commented out, configurable)
  • Automatic motor shutdown on communication loss
Returns
Never returns (infinite loop)
